Task types

Two workflow models for fast work and controlled approvals.

Both cards use the same structural system, which keeps the template maintainable while clearly differentiating urgency from governance.

Quick task

Direct execution for everyday work.

Best for short, routine assignments that can move immediately from creation to ownership.

  • HOD creates task with assignee, deadline, and priority.
  • Employee receives the task instantly.
  • Status updates remain visible throughout execution.
  • HOD reviews and closes the completed task.
Approval task

Executive sign-off for high-impact work.

Ideal for sensitive or cross-functional requests that need structured review before execution starts.

  • HOD submits a full task request for approval.
  • General Manager reviews, comments, and decides.
  • Approved work becomes active for the assignee.
  • The full decision trail stays available for audit.
How it works

The approval flow is clear from request to closure.

Every step is visible, timestamped, and easy to follow.

HOD

Task request created

The request captures title, detail, assignee, priority, and deadline before it enters review.

System

Approval queue triggered

The request moves into a pending state and the relevant approver is notified immediately.

General Manager

Decision with remarks

The approver accepts or rejects the request and can leave comments to preserve decision context.

Employee and HOD

Execution and closure

Once approved, the assignee works the task, submits completion, and the HOD closes it with full history intact.
